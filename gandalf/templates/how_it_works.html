How It Works

Process Flow

graph TD A[URL Input] --> B[URL Normalization] B --> C[Sitemap Discovery] C --> D[Content Crawling] D --> E[Content Analysis] E --> F[Category Matching] F --> G[Results Storage] style A fill:#f9f,stroke:#333 style G fill:#9f9,stroke:#333

1. URL Normalization

When you submit a URL, it's first normalized to ensure proper formatting:

  • Adds "https://" if protocol is missing
  • Removes trailing slashes
  • Validates URL format

2. Sitemap Discovery

The application attempts to find the website's sitemap:

  • Checks common sitemap locations (sitemap.xml, sitemap_index.xml)
  • Falls back to robots.txt if direct sitemap not found
  • Builds list of URLs to crawl

3. Content Crawling

Pages are crawled asynchronously for efficiency, with intelligent content extraction:

  • Respects robots.txt rules and handles rate limiting
  • Removes non-content elements:
    • Navigation menus and footers
    • Advertisements and promotional content
    • Sidebars and widgets
    • JavaScript and CSS code
  • Extracts meaningful content:
    • Main article text and headers
    • Important images and relevant links
    • Meta descriptions and titles
    • Structured data when available
  • Converts content to clean markdown format for analysis

4. Content Analysis

Extracted content is analyzed using advanced NLP:

  • Processes text to remove noise
  • Generates embeddings using SentenceTransformer
  • Prepares content for category matching

5. Category Matching

Content is matched against taxonomy categories:

  • Compares content embeddings with category embeddings
  • Calculates similarity scores for each tier
  • Ranks matches by similarity
  • Filters matches based on threshold

6. Results Storage

Results are stored and presented:

  • Saves detailed results in JSON format
  • Includes URLs, categories, and similarity scores
  • Provides downloadable results file
